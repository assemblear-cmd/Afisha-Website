# Instagram Story agent (локальный)

Агент, который на вашем компьютере регулярно постит Instagram-сторис по
ближайшим событиям Сантьяго с dondego.cl. Состоит из двух половин:

1. **Генератор** — `scripts/instagram-story/generate-story.mjs`: берёт
   ближайшее ещё не опубликованное событие из `GET /api/events`, рендерит
   фирменный сторис 1080×1920 (с обложкой события, если она доступна; иначе
   брендовый градиент) и печатает JSON с путём к PNG и данными события.
2. **Постер** — скилл `.claude/skills/instagram-story/SKILL.md`: локальная
   сессия Claude Code с расширением **Claude in Chrome** открывает ваш
   залогиненный Instagram, загружает PNG в сторис, добавляет линк-стикер на
   страницу события и после подтверждения публикации помечает событие как
   опубликованное.

Почему так: публикация сторис требует вашей залогиненной сессии Instagram —
она есть только в вашем локальном Chrome. Облачные сессии Claude до него не
дотягиваются, поэтому агент живёт на вашей машине.

---

## Требования

- Claude Code на компьютере, репозиторий склонирован
  (`/Users/skif/Documents/GitHub/Afisha-Website`).
- Chrome с расширением Claude in Chrome, подключённым к Claude Code, и
  залогиненным аккаунтом Instagram.
- `npm install` выполнен (генератору нужен `@playwright/test`; браузер он
  берёт системный Chrome, ничего не докачивает).

## Ручной запуск

В сессии Claude Code (из корня репозитория, с подключённым браузером):

```text
/instagram-story
```

Claude сгенерирует картинку, запостит сторис и отчитается со скриншотом.

Только сгенерировать картинку без постинга:

```bash
node scripts/instagram-story/generate-story.mjs
# PNG появится в scripts/instagram-story/out/, путь — в JSON-выводе
```

## Регулярный запуск

Самый надёжный вариант — цикл внутри открытой сессии Claude Code (расширение
подключено именно к интерактивной сессии):

```text
/loop 24h /instagram-story
```

Сессия раз в сутки будет генерировать и постить следующую сторис.

Вариант через cron (headless-запуск; расширение должно быть подключено, а
Chrome — открыт и залогинен):

```cron
# каждый день в 12:00 — постим сторис по ближайшему событию
0 12 * * * cd /Users/skif/Documents/GitHub/Afisha-Website && claude -p "/instagram-story" >> /tmp/dondego-story.log 2>&1
```

Для cron-запуска разрешите заранее нужные инструменты (браузер, Bash с
`node scripts/instagram-story/...`) в `.claude/settings.local.json`, иначе
headless-прогон упрётся в запрос разрешений.

## Как выбирается событие

- только Сантьяго (city/address содержит «Santiago»), только будущие;
- приоритет — ближайшие 14 дней (`STORY_LOOKAHEAD_DAYS`);
- уже опубликованные пропускаются — состояние в
  `scripts/instagram-story/.state/posted.json` (в git не попадает);
- когда всё опубликовано, агент сообщает `alreadyPosted: true` и без явной
  просьбы не репостит.

## Переменные окружения

| Переменная | По умолчанию | Зачем |
|---|---|---|
| `DONDEGO_API_BASE` | `https://dondego.cl` | источник событий (можно указать staging/локальный) |
| `STORY_LOOKAHEAD_DAYS` | `14` | горизонт «актуальности» события |
| `STORY_CHROME_PATH` | — | явный путь к Chrome/Chromium для рендера (иначе системный Chrome) |

## Ограничения и безопасность

- Скилл жёстко ограничен: одна сторис за прогон, никаких других действий в
  аккаунте; при любом неожиданном экране (логин, верификация) агент
  останавливается и зовёт вас.
- Instagram может менять веб-интерфейс создания сторис — в скилле заложены
  три пути (Create → Story, аватар «Your story +», прямой URL
  `/stories/create/`). Если сломаются все три, обновите шаг 3 скилла.
- Частые автоматические публикации с одного аккаунта Instagram может счесть
  подозрительными — раз в день безопасно, чаще не рекомендуем.
- Полностью серверная альтернатива без браузера — Instagram Graph API для
  Business/Creator-аккаунта (может публиковать сторис по токену; удобно
  связать с промо-услугой «Instagram story» из прайса платформы). Это
  отдельная фича — спрашивайте, если нужна.
